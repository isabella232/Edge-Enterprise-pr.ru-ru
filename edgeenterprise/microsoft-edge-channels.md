# <a name="overview-of-the-microsoft-edge-channels"></a>Обзор каналов в Microsoft Edge

Одно из преимуществ следующей версии Microsoft Edge в том, что Корпорация Майкрософт может регулярно предоставлять новые функции. Однако, как администратор, который Microsoft Edge пользователей в организации, вы можете получить больше контроля над тем, как часто пользователи получают эти новые функции. Корпорация Майкрософт предоставляет четыре варианта, называемых каналами, чтобы контролировать, Microsoft Edge обновляется с помощью новых функций. Ниже представлен обзор этих четырех вариантов.

Дополнительные сведения о поддержке для каждого канала читайте в Microsoft Edge [lifecycle](/deployedge/microsoft-edge-support-lifecycle)
  
> [!NOTE]
> Эта статья относится к Microsoft Edge версии 77 или более поздней.

## <a name="channel-overview"></a>Обзор каналов

|Канал|Основная цель|Частота добавления новых функций|Поддерживается?|
|:---:|---|:---:|:---:|
|[Stable](#stable-channel)|Широкое развертывание|~4 недели|Да|
|[Расширенная стабильность](#extended-stable-channel)|Параметр выпуска предприятия для Stable, выровненный к более длительному циклу выпуска |~8 недель|Да|
|[Beta](#beta-channel)|Показательная проверка внутри организации|~4 недели|Да|
|[Dev](#dev-channel)|Планирование и разработка|Еженедельно|Нет|
|[Canary](#canary-channel)|Новейшее содержимое|Ежедневно|Нет|

Канал обновления, который вы решили развернуть, зависит от нескольких факторов. Например, количество бизнес-приложений, которые используются, будет влиять на требования к тестированию при каждом Microsoft Edge обновлении. Чтобы вам было проще определиться, ознакомьтесь со следующей информацией о четырех каналах обновлений, доступных для Microsoft Edge.

### <a name="stable-channel"></a>Канал Stable

Стабильный канал предназначен для широкого развертывания в организации, и это канал, который должен быть в большинстве пользователей. Это самый стабильный из каналов и является результатом стабилизации набора функций, доступных в предыдущем выпуске бета-канала. Новые функции погрузка примерно каждые 4 недели. Обновления системы безопасности и исправления выпускаются по мере необходимости. Выпуск стабильного канала обслуживается до появления следующего выпуска в этом канале.

### <a name="beta-channel"></a>Канал Beta

Бета-канал предназначен для развертывания в представительной выборке пользователей. Это поддерживаемый выпуск, и каждый выпуск бета-версии будет обслужен до следующего выпуска. Этот канал предоставляет отличные возможности для проверки того, что все работает так, как ожидалось в вашей среде. Если вы найдете проблему, она может быть исправлена до публикации выпуска в стабильный канал. Новые функции погрузка примерно каждые 4 недели. Обновления системы безопасности и исправления выпускаются по мере необходимости.

### <a name="dev-channel"></a>Канал Dev

Канал Dev призван помочь в планировании и разработке новейших возможностей Microsoft Edge, но выпуски в нем отличаются более высоким качеством, чем в канале Canary. Этот канал — это ваша возможность получить ранний взгляд на то, что будет дальше, и подготовиться к следующему выпуску бета-версии.

### <a name="canary-channel"></a>Канал Canary

Выпуски в канале Canary выходят ежедневно и содержат самые новые функции. Если вы хотите получить доступ к самым новым инвестициям, они сначала появятся здесь. Из-за природы этой каденции со временем возникнут проблемы. Может потребоваться другой канал, установленный рядом, если вы используете выпуски Canary.

### <a name="extended-stable-channel"></a>Расширенный стабильный канал

В отличие от наших каналов предварительного просмотра (Canary, Dev и Beta), расширенный стабильный канал не доступен в качестве отдельного приложения-браузера. Этот канал является корпоративным вариантом выпуска для Microsoft Edge стабильного приложения, которое приведено в соответствие с более длительным, 8-недельным основным циклом выпуска. Это отличие от 4-недельного цикла основных выпусков для канала Stable. Хотя мы рекомендуем обновить Stable в 4-недельный цикл выпуска, extended Stable существует для более эффективного обслуживания организаций, для тестирования и проверки новых версий браузера может потребоваться более длинная временная шкала.

_8-недельный_ вариант выпуска "Extended Stable" для Microsoft Edge Stable предоставляет накопительные обновления функций, которые соответствуют ровным выпускам, начиная с Microsoft Edge 94. Обновления функций из выпусков с нечетными номерами будут упакованы и доставлены в рамках последующего выпуска с четными номерами. Например, если организация выбирает 8-недельный цикл выпуска "Расширенная стабильность" с Microsoft Edge 94, они получат последующие обновления функций с Microsoft Edge 96, Microsoft Edge 98 и так далее. Хотя обновления функций упакованы и доставляются с новыми версиями на основе выбранного цикла выпуска, важные исправления и исправления безопасности будут доставлены по мере необходимости независимо от выбранного варианта выпуска для поддержания безопасности браузера. Клиенты могут выбрать вариант расширенной стабильной версии в любое время, и он вступает в силу со следующей расширенной стабильной версией.

#### <a name="opting-in-to-the-extended-stable-release-cadence"></a>Выбор в расширенном курсиве стабильного выпуска

##### <a name="opting-in-to-extended-stable-on-windows-with-automatic-updates-recommended"></a>Выбор в расширенной стабильной Windows с автоматическими обновлениями (рекомендуется)

Если вы автоматически обновляете Microsoft Edge, вы можете использовать объекты групповой политики для выбора в расширенном курсе стабильного выпуска. [Дополнительные сведения о](/DeployEdge/configure-microsoft-edge#1-download-and-install-the-microsoft-edge-administrative-template) загрузке и установке последних шаблонов Microsoft Edge групповой политики.

1. Откройте редактор локальных групповых политик и перейдите в раздел _Конфигурация компьютера > Административные шаблоны > Центр обновления Microsoft Edge > Приложения > Microsoft Edge >_.
2. Выберите **переопределять целевой канал** и затем выберите **Включено.**
3. В **статье Параметры**выберите **"Расширенная стабильность"** из списка выпаданий политики.

Когда будет выпущено следующее обновление расширенного стабильного канала с номером версии выше установленного устройства, Microsoft Edge автоматически обновляется на расширенный стабильный канал. Строка версии `edge://settings/help` указывает, что вы работаете по другому каналу.

> [!NOTE]
> Выбор в расширенной стабильной версии вступает в силу при новом обновлении на расширенном стабильном канале с большим номером версии (основным или незначительным), чем установлено в настоящее время на вашем устройстве. Если вы работаете с последней версией Microsoft Edge и выбираете расширенную конюшню, она вступает в силу со следующим патчем или обновлением Microsoft Edge.
>
> По умолчанию Microsoft Edge не понизится. Если в настоящее время запущена нечетная версия Microsoft Edge Stable, то отказ от расширенной стабильной будет означать, что вы не будете получать обновления до следующего выпуска с четным номером Microsoft Edge.
>
> Если вы хотите убедиться, что все ваши устройства начинаются с определенной версии расширенной стабильной, вы можете развернуть эту конкретную версию Edge Stable в качестве MSI с включенной откатом. Например, если вы хотите начать с расширенной стабильной версии 94, но некоторые устройства уже обновлены до Stable 95, можно развернуть MSI edge 94 с включенной поддержкой отката. Дополнительные сведения о развертывании edge MSIs с включенной поддержкой отката см. в нашем руководстве [по откату.](/DeployEdge/edge-learnmore-rollback)

##### <a name="opting-in-to-extended-stable-on-windows-via-intune"></a>Выбор в расширенной стабильной Windows через Intune

Microsoft Edge административных шаблонов можно управлять аналогичными локальными объектами групповой политики из центра администрирования Microsoft Endpoint Manager. Следуйте нашему руководству по [настройке Microsoft Edge с Intune](/mem/intune/configuration/administrative-templates-configure-edge). 

Параметр **"Целевой**канал переопределения" можно найти в подвестрах Центр обновления Microsoft Edge >_приложений>Microsoft Edge_" . Он должен быть за набором **"Расширенная стабильность"** 

Когда будет выпущено следующее обновление расширенного стабильного канала с номером версии больше установленного устройства, Microsoft Edge автоматически будет обновляться на расширенном стабильном канале. Строка версии `edge://settings/help` указывает, что вы работаете по другому каналу.

##### <a name="opting-in-to-extended-stable-on-windows-via-configuration-manager"></a>Выбор в расширенной стабильной Windows с помощью диспетчера конфигурации

Дополнительные сведения о том Microsoft Edge как синхронизировать и утвердить обновления в [Configuration Manager,](/mem/configmgr/apps/deploy-use/deploy-edge#update-microsoft-edge) Microsoft Edge в нашем руководстве по обновлению.

Расширенные обновления stable распространяются в библиотеке программного обеспечения в категории продуктов Microsoft Edge, аналогично существующим обновлениям для каналов Stable, Beta и Dev. Однако, в отличие от бета-версии и dev, которые применяются к собственным приложениям браузера, расширенные стабильные обновления применяются к Microsoft Edge стабильному приложению. Поэтому для клиента Windows обновления, чтобы определить, следует ли применять стабильные или расширенные стабильные обновления, он проверяет состояние групповой политики "Целевой канал переопределения". Если политика не настроена или настроена на "Стабильный", будут применяться стабильные обновления. Если установлено "Расширенная стабильность", будут применяться расширенные обновления stable. Следуйте инструкциям, вышеуказанным для выбора в расширенной стабильной с помощью автоматических обновлений для инструкций по правильному набору групповой политики. 

## <a name="flighting-pre-release-channels-in-your-organization"></a>Flighting Pre-release Channels in your Organization

Политика группового переопределения целевого канала также может использоваться для бесперебойного полета каналов предварительного выпуска Microsoft Edge организации без использования пользователями второго приложения веб-браузера. Например, можно установить политику "Переопределения целевого канала" для "Бета" для представительного набора пользователей в организации. Когда эти пользователи Microsoft Edge, они будут запускать выпуск бета-канала, а не стабильный (возможно, даже не осознавая этого!). Этот параметр политики может дать вам раннее представление о том, как будет выполняться следующая версия Microsoft Edge в вашем предприятии, и поможет проверить, что все работает так, как ожидалось. Вы получите ранние сигналы от пользователей, столкнувшихся с любыми вопросами, и сможете убедиться, что они будут исправлены до публикации выпуска в стабильный канал. В рамках устранения проблемы пользователя строка версии сообщит вам, если канал пользователя отличается от канала Stable по `edge://settings/help` умолчанию.

> [!NOTE]
> Так как на основе каналов "Бета" и "Dev" Microsoft Edge основные номера версий больше, чем у канала "Stable", если вы берете обновление на канал "Бета" или "Dev" и хотите вернуться к [stable,](/DeployEdge/edge-learnmore-rollback) Microsoft Edge отката функция будет необходима. Простое настройка переопределения целевого канала обратно в Stable означает, что вы не получите никаких обновлений до тех пор, пока последний выпуск Stable не будет больше числа версий, чем версия Microsoft Edge, которая в настоящее время запущена на вашем устройстве.
